# 🔧 Fixing 404 Error - Step by Step

## Quick Fix (Most Common)

```bash
# Stop everything
docker-compose down

# Rebuild with no cache
docker-compose build --no-cache

# Start fresh
docker-compose up -d

# Wait 10 seconds
sleep 10

# Test
curl http://localhost
```

## Detailed Diagnosis

### Step 1: Verify Docker is Running

```bash
docker --version
docker-compose --version
docker ps
```

Expected: You should see version numbers and running containers.

### Step 2: Check Container Status

```bash
docker-compose ps
```

Expected output:
```
NAME                STATUS              PORTS
ledger_backend      Up                  0.0.0.0:8000->8000/tcp
ledger_db           Up (healthy)        0.0.0.0:3306->3306/tcp
ledger_frontend     Up                  0.0.0.0:80->80/tcp
```

If any container shows "Exit" status:
```bash
docker-compose logs [container-name]
```

### Step 3: Check Files in Frontend Container

```bash
# List files in nginx html directory
docker exec ledger_frontend ls -la /usr/share/nginx/html/

# You should see:
# index.html
# app.html  
# login.html (optional)
# test.html
# assets/ (directory)
```

If files are missing:
```bash
# Files aren't being copied - rebuild
docker-compose down
docker-compose build --no-cache frontend
docker-compose up -d
```

### Step 4: Check Nginx Configuration

```bash
# Test nginx config
docker exec ledger_frontend nginx -t

# View nginx config
docker exec ledger_frontend cat /etc/nginx/conf.d/default.conf
```

Expected: "nginx: configuration file /etc/nginx/nginx.conf test is successful"

### Step 5: Check Nginx Logs

```bash
# View frontend logs
docker-compose logs frontend

# Follow logs in real-time
docker-compose logs -f frontend
```

Look for errors like:
- "file not found"
- "permission denied"
- "failed to load"

### Step 6: Test Each Endpoint

```bash
# Test root
curl http://localhost

# Test specific files
curl http://localhost/index.html
curl http://localhost/app.html
curl http://localhost/test.html

# Test API health
curl http://localhost/health
curl http://localhost:8000/health
```

## Common Issues and Solutions

### Issue 1: Files Not Found (404)

**Cause:** Files not copied to container

**Solution:**
```bash
# Check your file structure
ls -la frontend/

# Should have:
# index.html
# app.html
# nginx.conf
# Dockerfile

# Rebuild
docker-compose build --no-cache frontend
docker-compose up -d frontend
```

### Issue 2: Port Already in Use

**Cause:** Another service using port 80

**Solution:**
```bash
# Option A: Find and stop conflicting service
sudo lsof -i :80
sudo kill -9 [PID]

# Option B: Change port in docker-compose.yml
# Edit frontend section:
ports:
  - "8080:80"

# Then access at http://localhost:8080
```

### Issue 3: Permission Denied

**Cause:** Wrong file permissions

**Solution:**
```bash
# Fix permissions in project
chmod -R 755 frontend/

# Rebuild
docker-compose build --no-cache frontend
docker-compose up -d
```

### Issue 4: Nginx Not Starting

**Cause:** Config file error

**Solution:**
```bash
# Check config file exists
ls frontend/nginx.conf

# Test config syntax locally if you have nginx installed
nginx -t -c frontend/nginx.conf

# View container logs
docker-compose logs frontend
```

### Issue 5: Container Keeps Restarting

**Cause:** Application crash or config error

**Solution:**
```bash
# See why it's failing
docker-compose logs frontend

# Check container status
docker-compose ps

# Force rebuild
docker-compose down -v
docker-compose up --build -d
```

## Complete Reset (Nuclear Option)

If nothing works, start completely fresh:

```bash
# 1. Stop and remove everything
docker-compose down -v

# 2. Remove all images
docker-compose down --rmi all

# 3. Prune docker system (optional - removes ALL unused containers/images)
docker system prune -a

# 4. Rebuild from scratch
docker-compose build --no-cache

# 5. Start fresh
docker-compose up -d

# 6. Wait for initialization
sleep 20

# 7. Check status
docker-compose ps

# 8. View logs
docker-compose logs

# 9. Test
curl http://localhost
```

## Still Getting 404?

### Check Browser

1. Clear browser cache (Ctrl+F5 or Cmd+Shift+R)
2. Try incognito/private mode
3. Try different browser
4. Check browser console (F12) for errors

### Check Network

```bash
# Can you reach Docker?
ping localhost

# Is port accessible?
telnet localhost 80

# Or use netcat
nc -zv localhost 80
```

### Check Docker Network

```bash
# Inspect network
docker network inspect ledger_network

# Check if containers can communicate
docker exec ledger_frontend ping backend
```

### Verify File Structure

Your project should look like this:

```
ledger-app/
├── docker-compose.yml
├── backend/
│   ├── Dockerfile
│   ├── main.py
│   ├── models.py
│   ├── schemas.py
│   ├── auth.py
│   ├── database.py
│   └── requirements.txt
└── frontend/
    ├── Dockerfile
    ├── nginx.conf
    ├── index.html       ← MUST EXIST
    ├── app.html         ← MUST EXIST
    ├── test.html
    └── assets/
        └── api.js
```
